Convex는 강력한 보장을 제공하는 소규모 조합 가능한 추상화 세트를 중심으로 설계되어, 팀 멤버나 LLM이 작성한 코드를 더 빠르게 작성할 수 있을 뿐만 아니라 읽고 유지 관리하기 쉽게 만듭니다. 주요 기능은 버그 없는 AI 생성 코드를 보장합니다:

1. **쿼리는 그저 TypeScript입니다** 데이터베이스 쿼리는 엔드투엔드 타입 안정성과 IDE 지원을 갖춘 순수 TypeScript 함수입니다. 이는 AI가 SQL로 전환하지 않고 대규모 TypeScript 코드 학습 세트를 사용하여 데이터베이스 코드를 생성할 수 있음을 의미합니다.
1. **동일한 작업에 더 적은 코드** Convex가 인프라와 보일러플레이트를 자동으로 관리하므로 작성할 코드가 적고, 따라서 잘못될 수 있는 코드도 적습니다.
1. **자동 반응성** 반응형 시스템은 데이터 종속성을 자동으로 추적하고 UI를 업데이트합니다. AI는 구독, WebSocket 연결 또는 복잡한 상태 동기화를 수동으로 관리할 필요가 없습니다—Convex가 이 모든 것을 자동으로 처리합니다.
1. **트랜잭션 보장** 쿼리는 읽기 전용이며 뮤테이션은 트랜잭션에서 실행됩니다. 이러한 제약 조건으로 인해 AI가 데이터를 손상시키거나 앱을 일관성 없는 상태로 만들 수 있는 코드를 작성하는 것이 거의 불가능합니다.

이러한 기능들이 함께 작동하여 AI가 비즈니스 로직에 집중할 수 있게 하는 동시에 Convex의 보장이 일반적인 실패 모드를 방지합니다. Convex에서 가장 잘 작동하는 모델에 대한 최신 정보는 LLM [리더보드](https://convex.dev/llm-leaderboard)를 확인하세요.

## Convex AI 규칙

AI 코드 생성은 따라야 할 규칙 세트를 제공할 때 가장 효과적입니다.

설치 지침은 다음 문서를 참조하세요:

- [Cursor](/ai/using-cursor.mdx#add-convex-cursorrules)
- [Windsurf](/ai/using-windsurf.mdx#add-convex-rules)
- [GitHub Copilot](/ai/using-github-copilot.mdx#add-convex-instructions)

다른 모든 IDE의 경우, 다음 규칙 파일을 프로젝트에 추가하고 변경 사항을 프롬프트할 때 참조하세요:

- [convex_rules.txt](https://convex.link/convex_rules.txt)

우리는 엄격한 평가를 사용하여 Convex에 대한 이러한 규칙의 품질을 지속적으로 개선하고 있습니다. [평가 저장소에 기여](https://github.com/get-convex/convex-evals)하여 도움을 주실 수 있습니다.

## 백그라운드 에이전트와 함께 Convex 사용하기

Jules, Devin, Codex, Cursor 백그라운드 에이전트와 같은 원격 클라우드 기반 코딩 에이전트는 CLI가 [에이전트 모드](/docs/cli/background-agents.mdx)일 때 Convex 배포를 사용할 수 있습니다. 이는 에이전트가 코드 생성, 코드 반복, 테스트 실행, 일회성 함수 실행을 수행할 수 있도록 하면서 이러한 원격 개발 환경에 필요한 권한을 제한합니다.

예를 들어 ChatGPT Codex를 위한 좋은 설정 스크립트는 다음과 같습니다:

```
npm i
CONVEX_AGENT_MODE=anonymous npx convex dev --once
```

또는

```
bun i
CONVEX_AGENT_MODE=anonymous bun x convex dev --once
```

이 명령은 바이너리를 다운로드하기 위해 "전체" 인터넷 액세스가 필요합니다.
