AI 코드 에디터인 [Cursor](https://cursor.com)는 Convex로 구축된 앱을 쉽게 작성하고 유지 관리할 수 있게 해줍니다. Convex에서 최상의 결과를 얻기 위해 Cursor를 설정하는 방법을 살펴보겠습니다.

## Convex `.cursor/rules` 추가

Cursor에서 최상의 결과를 얻으려면 모델별 `.mdc` 파일을 프로젝트의 `.cursor/rules` 디렉토리에 넣으세요.

## Convex MCP Server 설정

Convex CLI에는 내장된
[Convex Model Context Protocol](/ai/convex-mcp-server.mdx) (MCP) 서버가 함께 제공됩니다. Convex MCP 서버는 AI 코딩 에이전트에게 Convex 배포에 대한 액세스 권한을 부여하여 프로젝트를 쿼리하고 최적화할 수 있게 합니다.

### 빠른 설치

아래의 편리한 딥 링크를 클릭할 수 있습니다:

[![Install MCP Server](https://cursor.com/deeplink/mcp-install-dark.svg)](https://cursor.com/en/install-mcp?name=convex&config=eyJjb21tYW5kIjoibnB4IC15IGNvbnZleEBsYXRlc3QgbWNwIHN0YXJ0In0%3D)

### 수동 설치

Cursor를 시작하려면 "Cursor Settings > Tools & Integrations"를 열고 "New MCP Server"를 클릭한 다음, 열린 `mcp.json` 파일의 "mcpServers"에 "convex" 섹션을 추가하세요.

```json
{
  "mcpServers": {
    "convex": {
      "command": "npx",
      "args": ["-y", "convex@latest", "mcp", "start"]
    }
  }
}
```

[하나의 프로젝트에만](https://docs.cursor.com/en/context/mcp#configuration-locations) Convex MCP를 설치할 수도 있습니다.

서버를 추가한 후 "convex" 서버가 활성화되어 녹색으로 표시되는지 확인하세요
(NPM 패키지를 다운로드하는 동안 처음에는 1분 정도 걸릴 수 있습니다).

이제 다음과 같은 질문을 시작하세요:

- convex 스키마를 평가하고 개선 사항을 제안해주세요
- 이 앱의 공개 엔드포인트는 무엇인가요?
- `my_convex_function` 쿼리를 실행해주세요

## 팁과 요령

### Convex를 직접 설치하고 실행하기

Convex가 클라이언트 측 타입을 [자동으로 생성](https://docs.convex.dev/cli#run-the-convex-dev-server)하기 때문에 Convex를 계속 실행하는 것이 중요합니다. 이것이 없으면 에이전트가 생성한 쿼리 및 뮤테이션에 대한 타입에 액세스할 수 없기 때문에 린팅 루프에 갇힐 수 있습니다.

터미널 창에서 직접 Convex를 설치(`npm install convex`)하고 실행(`npx convex dev`)하는 것이 좋습니다.

### 요청을 작게 유지하기

에이전트 LLM을 사용할 때 최상의 결과는 변경하려는 양을 작게 유지하고 자주 git commit할 때 찾을 수 있습니다. 이를 통해 에이전트에게 제공하는 컨텍스트에 대해 더 구체적으로 지정할 수 있으며, 에이전트가 컨텍스트를 찾기 위해 많은 검색을 할 필요가 없습니다.

각 성공적인 프롬프트 또는 일련의 프롬프트 후에 변경 사항을 커밋하는 것이 좋습니다. 이렇게 하면 다음 프롬프트가 문제를 일으킬 경우 해당 지점으로 간단하게 롤백할 수 있습니다.

### `README.md` 업데이트 및 참조

에이전트는 프로젝트의 특정 비즈니스 목표에 대한 컨텍스트가 필요합니다.
읽는 파일에서 일부 세부 정보를 추론할 수 있지만 프로젝트가 성장함에 따라 더 어려워집니다. 프로젝트에 대한 일반 정보를 제공하면 에이전트에게 유용한 출발점이 됩니다.

각 프롬프트에 이 정보를 포함하는 대신 프로젝트 루트에 포괄적인 README.md 파일을 작성하고 참조하는 것이 좋습니다.

[일부 사람들](https://youtu.be/2PjmPU07KNs?t=145)은 제품 요구사항 문서(PRD) 작성을 옹호하며, 이는 더 복잡한 프로젝트에 좋은 아이디어일 수 있습니다.

### Convex 문서 추가

Convex 문서를 추가하면 앱을 구축할 때 Convex 기능을 구체적으로 참조할 수 있습니다.

**`Cursor Settings`** > **`Indexing & Docs`** > **`Docs`**에서 새 문서를 추가하고 URL "https://docs.convex.dev/home"을 사용하세요

![Chat UI](/img/cursor-with-convex/adding_convex_docs.webp)

그러면 Cursor가 LLM이 사용할 수 있도록 모든 Convex 문서를 인덱싱합니다.

![Chat UI](/img/cursor-with-convex/indexed_docs.webp)

그런 다음 `@Convex` 기호를 사용하여 프롬프트에서 해당 문서를 참조할 수 있습니다.

![Chat UI](/img/cursor-with-convex/reference_convex_docs.webp)
