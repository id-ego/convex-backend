클라우드에서 호스팅되는 Convex 개발 배포에 코드를 동기화하는 대신, 자신의 컴퓨터에서 실행되는 배포에 대해 개발할 수 있습니다. 로컬 배포에서도 Convex 대시보드를 사용할 수 있습니다!

## Convex의 배포에 대한 배경

각 Convex 배포에는 자체 데이터, 함수, 예약된 함수 등이 포함됩니다. 프로젝트에는 하나의 프로덕션 배포, 팀원당 최대 하나의 개발용 클라우드 배포, 그리고 잠재적으로 많은 일시적인 [프리뷰 배포](/production/hosting/preview-deployments.mdx)가 있습니다.

또한 자신의 컴퓨터에서 실행되는 배포를 사용하여 Convex로 개발할 수 있습니다. 배포가 로컬에서 실행되므로 코드 동기화가 더 빠르고, 함수 호출 및 데이터베이스 대역폭과 같은 리소스가 [Convex 플랜의 할당량](https://www.convex.dev/pricing)에 포함되지 않습니다.

기존 Convex 프로젝트에서 로컬 배포를 사용할 수 있으며, 프로젝트의 Convex 대시보드에서 배포를 볼 수 있습니다. Convex 계정 없이도 로컬 배포를 사용할 수 있으며, 로컬에서 실행되는 Convex 대시보드 버전으로 디버그하고 검사할 수 있습니다.

## 로컬 배포 사용하기

<BetaAdmonition feature="Local deployments" verb="are" />

로컬 배포를 사용하는 동안, 로컬 Convex 백엔드는 `npx convex dev` 명령의 하위 프로세스로 실행되며 해당 명령이 중지되면 종료됩니다. 즉, 이 로컬 배포에 대해 `npx convex run`과 같은 다른 명령을 실행하거나 프론트엔드가 이 배포에 연결하려면 `convex dev` 명령이 실행 중이어야 합니다.

로컬 백엔드의 상태는 `~/.convex/` 디렉토리에 저장됩니다.

### 익명 개발

계정을 만들지 않고도 로컬 배포를 사용하여 Convex로 개발할 수 있습니다. 프로덕션에 앱을 배포하거나 더 많은 Convex 기능을 사용하기 위해 계정을 만들고 싶을 때마다 `npx convex login`을 사용하여 로컬 배포를 계정에 연결할 수 있습니다.

### 기존 프로젝트에 대한 로컬 배포

기존 프로젝트에 대한 로컬 배포를 사용하려면 다음을 실행하세요:

```sh
npx convex dev --local --once
```

`npx convex dev --configure`를 실행하면 항상 로컬 배포 옵션이 제공됩니다. 다른 흐름은 일부 상황에서 클라우드 배포를 원한다고 가정할 수 있습니다. 예를 들어 이미 클라우드 개발 배포가 있는 프로젝트에 연결할 때가 그렇습니다.

## 로컬 배포 vs. 프로덕션

로컬 배포는 프로덕션 용도로 권장되지 않습니다. 이들은 개발 배포이므로 함수 결과에 대한 로그와 오류 응답에 대한 전체 스택 추적이 연결된 클라이언트로 전송됩니다.

프로덕션 애플리케이션을 실행하려면 Convex 클라우드에서 호스팅되는 프로덕션 배포를 사용할 수 있습니다. 프로덕션 배포에 대한 자세한 내용은 [여기](/production.mdx)에서 확인하세요.

또는 [오픈 소스 convex-backend 저장소](https://github.com/get-convex/convex-backend)를 사용하여 프로덕션 배포를 자체 호스팅할 수 있습니다.

### 비활성화

프로젝트에 대한 로컬 배포 사용을 중지하려면 다음을 실행하세요:

```sh
npx convex disable-local-deployments
```

클라우드 개발 배포와 각 로컬 개발 배포는 완전히 별도이므로 다른 데이터를 포함합니다. 배포를 전환할 때 데이터를 계속 사용하려면 데이터를 [내보내고 다시 가져오기](/database/import-export/import-export.mdx)를 원할 수 있습니다.

## 제한 사항

- **공개 URL 없음** - 클라우드 배포는 Twilio와 같은 서비스로부터 들어오는 HTTP 요청을 수신하기 위한 공개 URL이 있지만, 로컬 배포는 자신의 컴퓨터에서 HTTP 요청을 수신합니다. 마찬가지로, 사용자의 브라우저가 컴퓨터에 도달하는 방법을 알지 못하면 Convex WebSocket 연결로 웹사이트를 구동할 수 없습니다. 이러한 사용 사례에는 ngrok와 같은 프록시를 설정하거나 클라우드 배포를 사용하세요.

- **Node 액션에는 특정 Node.js 버전이 필요합니다** - Node.js 액션(파일에 `"use node;"`가 정의된 액션)을 실행하려면 Node.js 18이 설치되어 있어야 합니다. 이는 Node.js 액션이 AWS Lambda 함수에서 실행될 때 프로덕션에서 사용되는 Node.js 버전이기 때문입니다. 이를 해결하려면 [nvm](https://github.com/nvm-sh/nvm)을 설치 및 설정한 다음 Node.js 버전 18을 설치할 수 있습니다. 프로젝트의 나머지 부분에 Node.js 18을 사용할 필요는 없습니다.

- **Node.js 액션은 컴퓨터에서 직접 실행됩니다** - 일반 Node.js 서버와 마찬가지로 Node.js 액션에서 실행되는 코드는 무제한 파일시스템 액세스 권한을 가집니다. 쿼리, 뮤테이션 및 Convex 런타임 액션은 여전히 격리된 환경에서 실행됩니다.

- `npx convex dev` 명령이 다시 시작될 때마다 로그가 지워집니다.

- <a id="safari"></a> **Safari에서 대시보드 사용**: Safari는 [localhost에 대한 요청을 차단](https://bugs.webkit.org/show_bug.cgi?id=171934)하여 대시보드가 로컬 배포와 함께 작동하지 않도록 합니다. 로컬 배포를 사용하는 경우 다른 브라우저를 사용하는 것이 좋습니다.

- <a id="brave"></a> **Brave에서 대시보드 사용**: Brave는 [기본적으로 localhost에 대한 요청을 차단](https://brave.com/privacy-updates/27-localhost-permission/)하여 대시보드가 로컬 배포와 함께 작동하지 않도록 합니다. 다음 해결 방법을 사용할 수 있습니다:
  - `brave://flags/`로 이동
  - `#brave-localhost-access-permission` 플래그 활성화
  - Convex 대시보드로 돌아가기
  - URL 표시줄에서 **View Site Information**
    (<img src="/screenshots/brave-site-information.png" alt="View Site Information icon" width={24} style={{ verticalAlign: "middle" }} />)을 클릭한 다음 **Site settings** 클릭
  - **Localhost access** 설정을 **Allow**로 변경
