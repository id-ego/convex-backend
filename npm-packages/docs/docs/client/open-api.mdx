Convex는 Go, Java, C++를 포함한 많은 언어에 대한 명시적인 지원이 없습니다. 그러나 Convex 배포에서 [OpenAPI](https://swagger.io/specification/) 사양을 생성하여 현재 지원되지 않는 언어에 대한 타입 안전 클라이언트를 생성할 수 있습니다. 내부적으로 이는
[HTTP API](/http-api)를 사용합니다. 즉, 쿼리가 반응형/실시간이 아닙니다.

## 설정

<StepByStep>
  <Step title="Convex Helpers npm 패키지 설치하기">
    Open API 사양을 생성하는 CLI 명령이 포함된 `convex-helpers` 패키지를 설치하세요.

    ```sh
    npm install convex-helpers
    ```

  </Step>
  <Step title="OpenAPI 사양을 생성하는 명령 실행하기">
    이 명령을 실행하면 구성된 Convex 배포를 호출하고 그것을 기반으로 `convex-spec.yaml` 파일을 생성합니다. 명령에 `--help`를 전달하여 추가 플래그를 확인할 수 있습니다.

    ```sh
    npx convex-helpers open-api-spec
    ```

  </Step>
  <Step title="타입 안전 클라이언트 생성하기">
    별도의 도구를 사용하여 `convex-spec.yaml` 파일에서 클라이언트를 생성할 수 있습니다. 인기 있는 옵션으로는 [OpenAPI Tools](https://github.com/OpenAPITools/openapi-generator)와 [Swagger](https://swagger.io/tools/swagger-codegen/)가 있습니다.
    ```yaml
    # convex-spec.yaml
    openapi: 3.0.3
    info:
      title: Convex App - OpenAPI 3.0
      version: 0.0.0
      servers:
        - url: "{hostUrl}"
      description: Convex App API
      ...
    ```
  </Step>
</StepByStep>

## 제한사항

- 인수 및 반환 값 유효성 검사기는 필수가 아니지만 OpenAPI 사양의 타입을 풍부하게 만듭니다. 유효성 검사기가 정의되지 않은 경우 기본적으로 `v.any()`를 유효성 검사기로 사용합니다.
- Convex 배포 외부에서 내부 함수를 호출할 수 없습니다.
- 현재 `bigints` 또는 `bytes`를 지원하지 않습니다.
