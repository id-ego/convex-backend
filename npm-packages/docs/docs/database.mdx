Convex 데이터베이스는 관계형 데이터 모델을 제공하고, JSON과 유사한 문서를 저장하며, 스키마가 있거나 없이 사용할 수 있습니다. "그냥 작동하여" 사용하기 쉬운 인터페이스에서 예측 가능한 쿼리 성능을 제공합니다.

쿼리 및 뮤테이션 [함수](/functions.mdx)는 경량 JavaScript API를 통해 데이터를 읽고 씁니다. 설정할 것이 없으며 SQL을 작성할 필요도 없습니다. 앱의 요구사항을 표현하기 위해 JavaScript를 사용하기만 하면 됩니다.

아래에서 [테이블](#tables), [문서](#documents) 및 [스키마](#schemas)의 기본 사항을 학습한 다음 [데이터 읽기](/database/reading-data/reading-data.mdx) 및 [데이터 작성](/database/writing-data.mdx)으로 이동하세요.

앱이 더 복잡해짐에 따라 데이터베이스에서 더 많은 것이 필요해집니다:

- [문서 ID](/database/document-ids.mdx)를 사용한 관계형 데이터 모델링
- [인덱스](/database/reading-data/indexes/indexes.md)를 사용한 빠른 쿼리
- [페이지네이션 쿼리](/database/pagination.mdx)를 사용한 대규모 데이터셋 노출
- [스키마 정의](/database/schemas.mdx)를 통한 타입 안전성
- 데이터 [가져오기 및 내보내기](docs/database/import-export/import-export.mdx)를 통한 상호 운용성

## 테이블

Convex 배포에는 앱의 데이터를 보유하는 테이블이 포함되어 있습니다. 처음에는 배포에 테이블이나 문서가 없습니다.

각 테이블은 첫 번째 문서를 추가하는 즉시 생성됩니다.

```javascript
// `friends` 테이블이 존재하지 않습니다.
await ctx.db.insert("friends", { name: "Jamie" });
// 이제 존재하며, 문서가 하나 있습니다.
```

스키마를 미리 지정하거나 테이블을 명시적으로 생성할 필요가 없습니다.

## 문서

테이블에는 문서가 포함되어 있습니다. 문서는 JavaScript 객체와 매우 유사합니다. 필드와 값이 있으며, 그 안에 배열이나 객체를 중첩할 수 있습니다.

다음은 모두 유효한 Convex 문서입니다:

```json
{}
{"name": "Jamie"}
{"name": {"first": "Ari", "second": "Cole"}, "age": 60}
```

또한 다른 테이블의 다른 문서에 대한 참조를 포함할 수 있습니다. Convex에서 지원되는 타입에 대한 자세한 내용은 [데이터 타입](/database/types.md)을 참조하고, 해당 타입을 사용하여 데이터를 모델링하는 방법은 [문서 ID](/database/document-ids.mdx)를 참조하세요.

## 스키마

선택 사항이지만, 스키마는 데이터가 원하는 대로 정확히 보이도록 보장합니다. 간단한 채팅 앱의 경우 스키마는 다음과 같습니다:

```typescript
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

// @snippet start schema
export default defineSchema({
  messages: defineTable({
    author: v.id("users"),
    body: v.string(),
  }),
});
```

`v.any()`와 같은 타입을 사용하여 원하는 만큼 유연하게 선택하거나 `v.object()`를 정확하게 설명하여 원하는 만큼 구체적으로 선택할 수 있습니다.

스키마에 대한 자세한 내용은 [스키마 문서](/database/schemas.mdx)를 참조하세요.
