  Convex Deployment API는 베타 버전으로 공개적으로 사용할 수 있습니다. 사용 사례에 추가 기능이 필요한 경우 platforms@convex.dev로 문의해 주세요.

클라이언트에 애플리케이션별 기능을 노출하는 HTTP 엔드포인트와 달리, 관리 API 엔드포인트는 배포를 구성합니다(예: 환경 변수 수정).

## Authorization

Deployment Management API는 해당 배포에 대한 관리자 액세스 권한을 부여하는 키가 포함된 Authorization 헤더가 필요합니다.

[대시보드](https://docs.convex.dev/dashboard/deployments/deployment-settings#url-and-deploy-key) 또는 API 호출로 생성된 [Deployment keys](https://docs.convex.dev/cli/deploy-key-types#development-deploy-keys)를 이러한 API에 사용할 수 있습니다.

자신의 팀을 대신하여 Management API를 사용하는지 또는 구축한 Convex 통합의 사용자 팀을 대신하여 사용하는지에 따라 [Team Access Tokens](/platform-apis#managing-your-own-projects) 및 [OAuth Application Tokens](/platform-apis/oauth-applications)도 사용할 수 있습니다.

어떤 유형의 키든 앞에 `"Convex "` 문자열을 추가하세요.

```ts
const token = "ey...0=";
const response = await fetch(
  "https://happy-otter-123.convex.cloud/api/v1/list_environment_variables
  {
    headers: {
      Authorization: `Convex ${token}`,
    },
  },
);
console.log(await response.json());
```
