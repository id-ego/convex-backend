디버깅은 코드가 예상대로 작동하지 않는 이유를 파악하는 프로세스입니다.

## 개발 중 디버깅

개발 중에는 내장된 `console` API를 사용하여 함수 내부에서 무슨 일이 일어나고 있는지 이해할 수 있습니다:

```ts title="convex/myFunctions.ts"
import { mutation } from "./_generated/server";
import { v } from "convex/values";

export const mutateSomething = mutation({
  args: { a: v.number(), b: v.number() },
  handler: (_, args) => {
    console.log("Received args", args);
    // ...
  },
});
```

다음 메서드는 [기본 Convex 런타임](/functions/runtimes.mdx#default-convex-runtime)에서 사용할 수 있습니다:

- 지정된 심각도 수준으로 값 로깅:
  - `console.log`
  - `console.info`
  - `console.warn`
  - `console.error`
  - `console.debug`
- 스택 추적과 함께 로깅:
  - [`console.trace`](https://developer.mozilla.org/en-US/docs/Web/API/console/trace_static)
- 실행 시간 측정:
  - [`console.time`](https://developer.mozilla.org/en-US/docs/Web/API/console/time_static)
  - [`console.timeLog`](https://developer.mozilla.org/en-US/docs/Web/API/console/timelog_static)
  - [`console.timeEnd`](https://developer.mozilla.org/en-US/docs/Web/API/console/timeend_static)

Convex 백엔드는 또한 모든 성공적인 함수 실행과 함수에서 발생한 모든 오류를 자동으로 로깅합니다.

다음 위치에서 이러한 로그를 볼 수 있습니다:

1. [`ConvexReactClient`](/client/react.mdx)를 사용할 때 브라우저 개발자 도구 콘솔 창에서. 로그는 개발 배포에서 클라이언트로 전송되고 클라이언트가 브라우저에 로그를 기록합니다. 프로덕션 배포는 [클라이언트로 로그를 **전송하지 않습니다**](/functions/error-handling/error-handling.mdx#differences-in-error-reporting-between-dev-and-prod).
2. Convex 대시보드의 [로그 페이지](/dashboard/deployments/logs.md)에서.
3. 개발 중 [`npx convex dev`](/cli.md#tail-deployment-logs) 또는 로그만 출력하는 [`npx convex logs`](/cli.md#tail-deployment-logs)를 사용하는 터미널에서.

### 디버거 사용하기

테스트에서 함수를 실행할 수 있으며, 이 경우 `debugger;` 문을 추가하고 코드를 단계별로 실행할 수 있습니다. [테스트](/testing/convex-test.mdx#debugging-tests)를 참조하세요.

## 프로덕션에서 디버깅

프로덕션에서 문제를 디버깅할 때 옵션은 다음과 같습니다:

1. 기존 로깅 활용
2. 더 많은 로깅을 추가하고 프로덕션에 새 버전의 백엔드 배포

Convex 백엔드는 현재 제한된 수의 로그만 보존하며, Convex 팀이 내부 유지 관리 및 업그레이드를 수행할 때 언제든지 로그가 지워질 수 있습니다. 따라서 팀이 기록 로그와 클라이언트에서 로그한 추가 정보에 쉽게 액세스할 수 있도록 [로그 스트리밍 및 오류 보고](/production/integrations/integrations.mdx) 통합을 설정해야 합니다.

## 요청 ID로 관련 로그 찾기

사용자 또는 사용자가 경험하는 오류에 대한 적절한 로그를 찾기 위해 Convex는 개발 및 프로덕션 모두에서 모든 예외 메시지에 다음 형식으로 요청 ID를 포함합니다: `[Request ID: <request_id>]`.

요청 ID를 복사하여 Convex 대시보드에 붙여넣어 해당 요청으로 시작된 함수의 로그를 볼 수 있습니다. 자세한 내용은 [대시보드 로그 페이지](/dashboard/deployments/logs.md#filter-logs)를 참조하세요.
