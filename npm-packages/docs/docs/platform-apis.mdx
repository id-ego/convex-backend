# Platform APIs

<Admonition type="info">
  Convex Platform APIs는 베타 버전으로 공개적으로 제공됩니다.
  사용 사례에 추가 기능이 필요한 경우 platforms@convex.dev로 문의하세요.
</Admonition>

이 가이드는 자신의 계정에서 여러 Convex 프로젝트를 조정하거나
사용자 계정에서 프로젝트를 관리하려는 제품을 위한 것입니다.
이러한 API는 [Bloom](https://bloom.diy/) 또는 [A0](https://a0.dev/)와 같은
AI 앱 빌더가 가장 자주 사용합니다.

이 가이드는 Convex 클라우드 계층 구조(팀, 프로젝트 및 배포)와
[개발 워크플로우](/understanding/workflow)에 대한 충분한 이해를 전제로 합니다.

## 자신의 프로젝트 관리

이는 소유한 팀의 컨텍스트에서 프로그래밍 방식으로 프로젝트, 배포를 생성하고
코드를 푸시하는 것을 의미합니다.

자신의 팀에서 프로젝트를 관리하려면
[Team Settings](https://dashboard.convex.dev/team/settings/access-tokens)에서
팀 범위 토큰과 ID를 가져와야 합니다.

<Admonition type="caution">
  이 토큰은 토큰을 검색할 때 Convex 대시보드에 로그인한 팀 멤버가 소유합니다.

이는 이 사용자가 이러한 토큰을 사용하여 생성된 모든 개발 배포를 소유한다는 것을 의미합니다.
이 사용자가 팀을 떠나면 팀에서 모든 개발 배포도 삭제됩니다.

팀 멤버로 추가된 별도의 서비스 계정을 만드는 것이 좋습니다.
이 서비스 계정으로 로그인한 후 토큰을 검색하세요.

</Admonition>

## 사용자 프로젝트 관리

이는 사용자가 자신의 Convex 팀 또는 프로젝트를 관리할 수 있도록
귀하의 제품에 권한을 부여하는 것을 의미합니다.

이를 위해서는 사용자가 제품에 필요한 권한을 부여할 수 있도록
OAuth 2.0 애플리케이션을 생성해야 합니다.

[OAuth Applications](/platform-apis/oauth-applications) 가이드를 따라
OAuth 애플리케이션을 생성하고 관련 토큰을 요청하세요.

## 배포에 코드 푸시

배포 작업은 주로 기존 Convex CLI를 사용하여 스크립트로 작성해야 합니다.
Convex CLI는 코드 번들링, 응답 적절한 처리 등 많은 무거운 작업을 관리합니다.

여기의 예제는 앱 빌딩 프로세스를 구동할 수 있는 쉘 및 파일 시스템 액세스가 있는
컨테이너에서 작업하고 있다고 가정합니다. 프론트엔드 코드를 생성하는 경우
이미 이것이 있을 것입니다.

[`CONVEX_DEPLOY_KEY`](/docs/cli/deploy-key-types.mdx)를
[Create deploy key](/management-api/create-deploy-key) API에서 반환된 값으로 설정합니다.

### 개발 Convex 백엔드에 코드 푸시

```bash
CONVEX_DEPLOY_KEY="YOUR_DEPLOY_KEY" npx convex dev --once
```

### 프로덕션 Convex 백엔드에 코드 푸시

```bash
CONVEX_DEPLOY_KEY="YOUR_DEPLOY_KEY" npx convex deploy
```

전체 명령 목록을 보려면 [CLI 문서](/cli)를 참조하세요.
