Convex에서 구축하는 데 _무제한의_ 즐거움을 드리고 싶지만, 엔지니어링 실무상 몇 가지 제한이 필요합니다. 이 페이지는 Convex 생태계의 현재 제한 사항을 설명합니다.

이러한 제한 사항 중 많은 부분은 시간이 지남에 따라 더 완화될 것입니다. 제한 사항이 애플리케이션에 문제가 되는 경우 연락해 주세요.

제한은 별도로 명시되지 않는 한 팀당 적용됩니다.

## 팀

|            | Free/Starter | Professional             |
| ---------- | ------------ | ------------------------ |
| 개발자     | 1-6          | 멤버당 월 $25            |
| 프로젝트   | 20           | 100                      |

### 제약 사항

- 테이블 및 인덱스 이름은 유효한 식별자여야 하며 밑줄로 시작할 수 없습니다.

## 문서

별도로 명시되지 않는 한 문서당 및 중첩된 `Object`에 적용됩니다.

|                     |               | 참고 사항                                                    |
| ------------------- | ------------- | ------------------------------------------------------------ |
| 크기                | 1 MiB         |                                                              |
| 필드                | 1024          | 필드/키의 수                                                 |
| 필드 이름 길이      | 64자          | 중첩된 `Object` 키는 최대 1024자까지 가능합니다.             |
| 필드 중첩 깊이      | 16            | 객체와 배열이 중첩될 수 있는 횟수, 예: `[[[[]]]]`            |
| 배열 요소           | 8192          |                                                              |

### 제약 사항

- 필드 이름은 제어 문자가 아닌 영숫자 ASCII 문자와 밑줄만 포함해야 하며 알파벳 문자나 밑줄로 시작해야 합니다.
- 문서는 시스템 제공 `_id` 및 `_creationTime` 필드를 제외하고 밑줄로 시작하는 최상위 필드를 포함할 수 없습니다.
- 문자열은 짝을 이루지 않은 서로게이트가 없는 유효한 유니코드 시퀀스여야 합니다.

## 동시 함수 실행

각 함수 유형에 대해 동시에 실행할 수 있는 함수의 수입니다.

|                | Free/Starter | Professional<sup>\*</sup> |
| -------------- | ------------ | ------------------------- |
| 쿼리           | 16           | 256                       |
| 뮤테이션       | 16           | 256                       |
| V8 액션        | 64           | 256                       |
| Node 액션      | 64           | 1000                      |
| HTTP 액션      | 16           | 128                       |
| 예약된 작업    | 10           | 300                       |

<sup>*</sup> Professional 플랜 고객의 경우 사례별로 제한을 완화할 수 있습니다. 대시보드를 통해 지원 메시지를 보내 필요 사항을 논의하세요. 일반적으로 제품에 트래픽이 급증하는 경우에만 필요합니다.

이러한 제한 사항이 맞지 않는 경우, [알려주세요](https://convex.dev/community)!

리소스 제한에 대해서는 [플랜 및 가격 페이지](https://www.convex.dev/pricing)를 참조하세요. 무료 플랜에서 이러한 제한에 도달하면 더 많은 삽입 또는 업데이트를 커밋하려는 새 뮤테이션이 실패할 수 있습니다. 유료 플랜에는 하드 리소스 제한이 없으며 수십억 개의 문서와 TB의 스토리지로 확장할 수 있습니다.
