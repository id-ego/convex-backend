특정 요구 사항이 있거나 Convex의 보장으로 아직 충족되지 않는 프로젝트를 구축하려는 경우 [문의해 주세요](mailto:support@convex.dev).

## 보장

공식 Convex 서비스 약관, 개인정보 보호정책 및 고객 계약은 [공식 약관](https://www.convex.dev/legal/tos)에 설명되어 있습니다. 아직 공식 약관에 나열된 것 이외의 계약상 합의는 없으며, 이 문서 내의 논의는 이러한 약관에 대한 수정을 구성하지 않습니다.

Convex는 항상 지속적으로 개발 중이며 향후 릴리스에서는 새 버전으로 업그레이드하기 위해 코드 변경이 필요할 수 있습니다. Convex 1.0 이상에서 개발된 코드는 그대로 계속 작동합니다. 향후 중대한 변경이 필요한 경우 팀에 직접 연락하여 충분한 사전 통지를 제공할 것입니다.

Convex의 모든 사용자 데이터는 저장 시 암호화됩니다. 데이터베이스 상태는 여러 물리적 가용 영역에 걸쳐 내구성 있게 복제됩니다. 정기적인 주기적 및 증분 데이터베이스 백업이 수행되며 99.999999999%(11 9's) 내구성으로 저장됩니다.

Convex 배포의 가용성은 99.99%(4 9's)를 목표로 하지만 사전 통지 없이 유지 관리를 위한 다운타임이 발생할 수 있습니다. 물리적 중단은 배포의 가용성에 영향을 줄 수 있지만 Convex에 저장된 데이터의 내구성에는 영향을 주지 않습니다.

## 베타 기능

이 문서에서 **베타**로 표시된 기능은 아직 개발 중입니다. 프로덕션에서 사용할 수 있지만 향후 API가 변경될 수 있으며, 새 버전의 Convex NPM 패키지 및 기타 Convex 클라이언트 라이브러리로 업그레이드할 때 추가 작업이 필요할 수 있습니다.

## 향후 기능

Convex는 아직 매우 활발하게 개발 중이며 여기에 우리가 주목하고 있는 누락된 기능 중 일부를 나열합니다. [Convex Discord 커뮤니티](https://convex.dev/community)에서 여러분의 요구 사항에 대해 더 듣고 싶습니다.

### 인가

Convex는 현재 사용자 신원을 확인하는 [_인증 프레임워크_](/auth.mdx)가 있습니다. 향후 개발자가 사용자가 액세스할 수 있는 데이터를 정의할 수 있는 _인가 프레임워크_를 추가할 계획입니다.

현재로서는 쿼리 및 뮤테이션 내에서 수동 인가 검사를 구현할 수 있지만 향후 더 포괄적이고 완벽한 솔루션을 기대해 주세요.

### 원격 측정

현재 대시보드는 기본 메트릭만 제공합니다. 대규모 사이트는 로그와 메트릭을 분류하고 알림과 같은 기능을 지원하는 보다 완전한 관찰 가능성 시스템에 통합해야 합니다.

Convex는 결국 제3자가 수집할 수 있는 형식으로 배포 데이터를 게시하는 방법을 제공할 것입니다.

### 분석 / OLAP

Convex는 주로 앱의 모든 실시간 구현(OLTP) 요구 사항을 처리하도록 설계되었습니다. 분석(OLAP) 사용 사례의 요구 사항을 해결하는 데 필요한 복잡한 쿼리 및 대규모 테이블 스캔에는 적합한 솔루션이 아닙니다.

Convex는 Convex 데이터를 외부 분석 시스템으로 내보내기 위한 [Fivetran 및 Airbyte 커넥터](/production/integrations/streaming-import-export.md)를 제공합니다.

### 브라우저 지원

Convex는 아직 공식 브라우저 지원 정책이 없지만 상당한 [사용량](https://caniuse.com/usage-table)을 보이는 대부분의 최신 브라우저를 지원하기 위해 노력하고 있습니다.
