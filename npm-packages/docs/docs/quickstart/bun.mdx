Bun 프로젝트에서 Convex 데이터를 쿼리하는 방법을 알아보세요.

시점 쿼리 대신 구독에 대한 지침은 [Bun 참고사항](/client/javascript/bun.mdx)을 참조하세요.

# Bun과 함께 Convex 사용하기

<StepByStep>
  <Step title="새 Bun 프로젝트 생성하기">
    Bun 프로젝트를 위한 새 디렉토리를 생성하세요.

    ```sh
    mkdir my-project && cd my-project && bun init -y
    ```

  </Step>
  <Step title="Convex 클라이언트 및 서버 라이브러리 설치하기">
    `convex` 패키지를 설치하세요.

    ```sh
    bun add convex
    ```

  </Step>
  <Step title="Convex 개발 배포 설정하기">
    다음으로, `bunx convex dev`를 실행하세요. 이것은 GitHub로 로그인하고, 프로젝트를 생성하고, 프로덕션 및 배포 URL을 저장하라는 메시지를 표시합니다.

    또한 백엔드 API 함수를 작성할 수 있는 `convex/` 폴더를 생성합니다. `dev` 명령은 계속 실행되면서 함수를 클라우드의 개발 배포와 동기화합니다.


    ```sh
    bunx convex dev
    ```

  </Step>

  <Step title="데이터베이스용 샘플 데이터 생성하기">
    새 터미널 창에서 샘플 데이터가 포함된 `sampleData.jsonl` 파일을 생성하세요.

    <Snippet
      source={sampleData}
      title="sampleData.jsonl"
    />

  </Step>

  <Step title="데이터베이스에 샘플 데이터 추가하기">
    이제 프로젝트가 준비되었으므로 `import` 명령을 사용하여 샘플 데이터가 포함된 `tasks` 테이블을 Convex 데이터베이스에 추가하세요.

    ```
    bunx convex import --table tasks sampleData.jsonl
    ```

  </Step>

  <Step title="데이터베이스 쿼리 노출하기">
    `convex/` 폴더에 데이터를 로드하는 쿼리 함수가 포함된 새 파일 `tasks.js`를 추가하세요.

    이 파일에서 쿼리 함수를 내보내면 파일과 내보내기 이름으로 명명된 API 함수 `api.tasks.get`이 선언됩니다.

    <Snippet
      source={tasks}
      title="convex/tasks.js"
    />

  </Step>

  <Step title="스크립트를 백엔드에 연결하기">
    새 파일 `index.ts`에서 개발 환경의 URL을 사용하여 `ConvexClient`를 생성하세요.

    <Snippet
      source={script}
      title="index.ts"
    />

  </Step>

  <Step title="스크립트 실행하기">
    같은 디렉토리에서 스크립트를 실행하고 터미널에 작업 목록이 출력되는 것을 확인하세요.

    ```sh
    bun index.ts
    ```

  </Step>

</StepByStep>

전체 [Bun 문서](/client/javascript/bun.mdx)를 참조하세요.
