지속적 통합을 통해 팀의 모든 구성원의 변경 사항을 통합하고 원격 머신에서 자동으로 테스트하여 팀이 빠르게 움직일 수 있습니다.

## GitHub Actions에서 테스트하기

[GitHub](https://docs.github.com/en/actions)을 사용하는 경우 테스트 스위트를 실행하기 위한 [CI](https://docs.github.com/en/actions/automating-builds-and-tests/about-continuous-integration) 워크플로우를 쉽게 설정할 수 있습니다:

```yaml title=".github/workflows/test.yml"
name: Run Tests

on: [pull_request, push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run test
```

이 파일을 커밋하고 레포지토리에 푸시하면, GitHub은 풀 리퀘스트를 생성하거나 새 커밋을 푸시할 때마다 `npm run test`를 실행합니다.
