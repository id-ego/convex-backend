[`convex-test`](/testing/convex-test.mdx) 대신 [오픈소스 버전의 Convex 백엔드](https://github.com/get-convex/convex-backend)를 사용하여 함수를 테스트할 수 있습니다.

## 시작하기

[이 가이드](https://stack.convex.dev/testing-with-local-oss-backend)를 따라 지침을 확인하세요.

백엔드의 JS 모의 객체를 사용하는 `convex-test`와 비교하여, 실제 백엔드에 대해 테스트를 실행하면 다음과 같은 장점이 있습니다:

- 로컬 백엔드를 최신 상태로 유지하는 한, 테스트는 Convex 프로덕션과 동일한 코드에 대해 실행됩니다.
- 인수, 데이터, 쿼리 크기에 대한 제한이 적용됩니다.
- 데이터 가져오기를 통해 대규모 테스트 데이터셋을 부트스트랩할 수 있습니다.
- 백엔드 로직과 함께 클라이언트 코드를 테스트할 수 있습니다.

## 제한사항

로컬 백엔드에 대한 테스트는 다음과 같은 단점도 있습니다:

- 로컬 백엔드 설정이 필요하며, 더 복잡합니다.
- 시간을 제어할 수 없으며 예약된 함수는 예정대로 실행됩니다.
- Cron도 [`IS_TEST`](https://stack.convex.dev/testing-with-local-oss-backend#setting-up-a-local-backend)를 통해 비활성화하지 않는 한 실행됩니다.
- `fetch` 호출을 모의 객체로 만들 방법이 없습니다.
- 의존성이나 코드베이스의 일부를 모의 객체로 만들 방법이 없습니다.
- 무작위성을 제어할 방법이 없습니다(테스트가 결정론적이지 않을 수 있음).
- 테스트 내에서 환경 변수 값을 설정할 방법이 없습니다.

모의 Convex 백엔드를 사용하여 JS로 함수를 테스트하려면 [convex-test](/testing/convex-test.mdx)를 확인하세요.

## CI

공유 원격 머신에서 테스트를 실행하려면 [Continuous Integration](/testing/ci.mdx)을 참조하세요.
